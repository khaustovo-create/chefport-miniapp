# Review после заказа — User Flow

```
┌─────────────────────────────────────────────────────────────┐
│  POST /api/order — заказ оформлен                           │
│  orders: { id, user_id, created_at, ... }                   │
└───────────────────────┬─────────────────────────────────────┘
                        │
                        │  через 24 часа
                        ▼
┌─────────────────────────────────────────────────────────────┐
│  SCHEDULER                                                  │
│  SELECT orders WHERE created_at <= now-24h                  │
│                   AND review_requested = 0                  │
│                   AND user_id IS NOT NULL                   │
└───────────┬─────────────────────────┬───────────────────────┘
            │                         │
      user_id есть             user_id = NULL
            │                         │
            │                         ▼
            │                   пропустить
            │                   (анонимный заказ,
            │                    написать некуда)
            ▼
┌───────────────────────────────────────┐
│  БОТ → покупателю:                    │
│  «Ваш заказ доставлен? Оцените        │
│   от 1 до 5 ⭐»                        │
│                                       │
│  reviews: { order_id, user_id,        │
│    requested_at, status='pending' }   │
└───────────┬───────────────────────────┘
            │
            ├─────────────────────────────────────────────────┐
            │                                                 │
     ответил (≤ 48 ч)                             не ответил (> 48 ч)
            │                                                 │
            ▼                                                 ▼
   покупатель пишет                              reviews: { status='no_response',
   сообщение боту                                  rating=null, text=null }
            │
            ├──── не число / вне 1–5 ────────────────────────┐
            │                                                 │
            │                                          БОТ: «Пожалуйста,
            │                                          отправьте число
            │                                          от 1 до 5»
            │                                          (до 3 попыток,
            │                                           потом — no_response)
            │
     число 1–5 получено
            │
            ▼
┌───────────────────────────────────────┐
│  БОТ → покупателю:                    │
│  «Спасибо! Напишите пару слов         │
│   или отправьте /skip»                │
└───────────┬───────────────────────────┘
            │
            ├──────────────────────────────────────────────────┐
            │                                                  │
     написал отзыв                                  /skip или молчание (24 ч)
            │                                                  │
            └────────────────────┬─────────────────────────────┘
                                 │
                                 ▼
                ┌────────────────────────────────────┐
                │  reviews: {                        │
                │    order_id, user_id,              │
                │    rating      (1–5),              │
                │    text        (или null),         │
                │    status    = 'done',             │
                │    created_at                      │
                │  }                                 │
                └───────────────┬────────────────────┘
                                │
                                ▼
               ┌────────────────────────────────────┐
               │  БОТ → покупателю: «Спасибо! 🙏»   │
               └───────────────┬────────────────────┘
                               │
                               ▼
               ┌────────────────────────────────────┐
               │  БОТ → админу (опционально):       │
               │  «⭐ Новый отзыв                   │
               │   Заказ: #ms_order_name            │
               │   Оценка: 4/5                      │
               │   Текст: "..."»                    │
               └────────────────────────────────────┘
```

---

**Что нужно для реализации:**

| Что | Где |
|---|---|
| Новая таблица `reviews` | `db.js` |
| Флаг `review_requested` в `orders` | `db.js` (новая колонка или отдельная таблица) |
| Планировщик (cron / setInterval) | новый модуль или в `bot.js` |
| Обработчик состояния диалога | `bot.js` — нужен простой state machine (ждём рейтинг / ждём текст) |
| Команда `/reviews` для админа | `bot.js` |

**Главный риск:** `user_id` в таблице `orders` сейчас опциональный — если пользователь зашёл без Telegram-контекста, написать ему невозможно. Схема это учитывает через ветку `user_id = NULL`.
